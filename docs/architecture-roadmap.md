# Architecture Roadmap

当前版本保持为单 locale、RSS-only、单入口运行模式：

- 数据源固定来自 sources.yaml 中的 RSS 列表
- 单次运行只处理一个 locale，由 config.yaml 顶层 locale 控制
- 运行入口固定为 python -m src.main

## 后续如果重新支持多数据源

重新引入前先满足下面三个条件：

1. 出现至少第二种真实数据源，而不是为“以后可能会有”预留。
2. 不同数据源在抓取协议、认证或限流策略上确实不同，无法继续用统一 RSS 抓取层表达。
3. 主流程能从配置直接驱动，而不是先定义注册表再绕回现有实现。

推荐落地顺序：

1. 先定义统一的抓取结果模型和最小 fetch 接口。
2. 在主流程中显式遍历 source 类型，而不是依赖隐式导入注册。
3. 只有当 source 类型超过 2 到 3 种时，再评估是否需要注册表或插件发现。

## 后续如果重新支持多 locale 管道

重新引入前先满足下面两个条件：

1. 不同 locale 确实需要不同收件人、模板或数据源，而不是只改翻译文本。
2. 已经明确单次运行要串行发送还是并行发送，以及失败隔离策略。

推荐落地顺序：

1. 先把单 locale 的配置对象抽成可复用任务对象。
2. 在 main 中显式构建 locale task 列表并逐个执行。
3. 最后再扩展模板、收件人和 sources 文件覆盖能力。

## 约束

- 不再恢复“脚本运行/包运行双模式兼容”。
- 新抽象必须先接入主流程，再写进 README。
- 如果只是为了减少文件长度，不要再次拆回 mixin 结构；优先用单类内分区或少量组合对象。
